§AstraWeave Core

Central shared types and simulation vocabulary for the AstraWeave engine.

This crate defines the canonical data types used across all engine subsystems:

  • schema — WorldSnapshot, PlayerState, CompanionState, EnemyState, PlanIntent, ActionStep, and other shared gameplay types.
  • tool_sandbox / tool_vocabulary — ToolRegistry and ToolSpec for AI action validation (the “tool sandbox” pattern).
  • capture_replay — Deterministic capture and replay infrastructure.
  • perception — AI perception helpers (astar_path, find_cover_positions, los_clear).
  • validation — Configuration validation traits.
  • metrics — Telemetry and performance counters.

§Architecture

The core crate sits at the bottom of the dependency graph. Every other crate (AI, physics, gameplay, rendering) depends on astraweave-core for shared type definitions, ensuring a single source of truth for the AI-native loop:

Perception → Reasoning → Planning → Action
    ↓            ↓           ↓          ↓
WorldSnapshot  AI Model   PlanIntent  Tool Validation
